# Guide 4 : Configuration du Poste Mac pour le Traitement Terrain

# Ce guide détaille les étapes pour configurer macOS, installer les dépendances requises (Docker, clés SSH), et préparer l'arborescence des dossiers afin que le script `mission_sync.sh` fonctionne en un seul clic au retour d'exploration.

## 1. Installation de Docker Desktop sur Mac

# Le traitement des données (DLIO, Rosbag) s'exécute dans un conteneur Linux isolé sur ton Mac.

# 1. Télécharge et installe **Docker Desktop pour Mac** (sélectionne bien la version **Apple Silicon** si tu as un processeur M1/M2/M3/M4).
# 2. Lance Docker Desktop et va dans les **Paramètres** (roue crantée) > **Resources** > **Virtual Filesystem** :
#    * Assure-toi que Docker a l'autorisation d'accéder au dossier `Users/samuel/desktop`.
# 3. Laisse l'application Docker Desktop tourner en arrière-plan.

## 2. Configuration des Clés SSH (Connexion sans mot de passe)
# Pour que le script `mission_sync.sh` puisse aspirer automatiquement les fichiers de Chinook via `rsync` sans bloquer le script en te demandant le mot de passe à chaque fois, il faut copier ta clé SSH publique sur le Raspberry Pi.

# Ouvre un terminal sur ton Mac et tape :

# 1. Génère une clé SSH sur ton Mac (si tu n'en as pas déjà une - appuie sur Entrée pour tout laisser par défaut)
ssh-keygen -t rsa -b 4096

# 2. Copie la clé sur Chinook (il va te demander le mot de passe 'chinook' une dernière fois)
ssh-copy-id samuel@chinook.local
# Fais un test : tape ssh samuel@chinook.local (ou avec ton $USER et le nom de ton Rpi.Local). Si tu te connectes instantanément sans qu'on te demande de mot de passe, c'est gagné !

# 3. Création de l'Arborescence des Dossiers sur le Mac
# Le script et le conteneur Docker s'appuient sur une structure de dossiers stricte sur ton Bureau. Crée-la manuellement ou via ces commandes :

mkdir -p /Users/${USER}/desktop/Expedition_Data/raw
mkdir -p /Users/${USER}/desktop/Expedition_Data/results
mkdir -p /Users/${USER}/desktop/Expedition_Data/ros2_ws

# 📁 Rôle des dossiers :
# raw/ : Zone de réception. C'est ici que rsync télécharge les Rosbags bruts (cave_bag_...) depuis Chinook.

# results/ : Zone de sortie. C'est ici que Docker enregistre le Rosbag traité final (_result) et que ton script Python génère ton fichier de topographie .tro.

# ros2_ws/ : C'est le "pont" (volume partagé) avec Docker. Ton script lidar_to_vtopo.py doit être placé directement dans le sous-dossier results/ de cette arborescence.

# 4. Autoriser AppleScript à renommer tes Terminaux
# Le script mission_sync.sh utilise des commandes osascript pour ouvrir et renommer automatiquement tes fenêtres de travail (T1 à T5).

# La première fois que tu vas lancer le script, macOS va afficher une alerte de sécurité te demandant : "L'application Terminal souhaite contrôler l'application Terminal/Système".
# 👉 Clique impérativement sur "Autoriser" ou "OK".

# Si tu as cliqué sur refuser par erreur, va dans :
# Réglages Système > Confidentialité et sécurité > Automatisation et coche la case sous Terminal.

# 5. Rendre le script exécutable et le lancer
# Place ton fichier mission_sync.sh à la racine de ton dossier de travail (par exemple dans /Users/samuel/desktop/Expedition_Data/).

# Ouvre ton terminal Mac, déplace-toi dans le dossier et donne-lui les droits d'exécution :

cd /Users/samuel/desktop/Expedition_Data
chmod +x mission_sync.sh

# Pour lancer ton usine de traitement au retour d'une sortie en grotte, connecte Chinook (ou le RPi avec le nom que tu lui aura donné) au même réseau Wi-Fi que ton Mac (ou via un câble Ethernet) et tape simplement :


./mission_sync.sh
