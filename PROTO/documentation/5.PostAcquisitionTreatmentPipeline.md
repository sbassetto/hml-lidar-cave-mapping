# Manuel d'utilisation du Pipeline Spéléologique LiDAR

Ce document décrit le flux de travail permettant de traiter les données LiDAR brutes, de les assembler géométriquement, puis d'en extraire la topographie complète de la cavité.

## 1. Dépendances et Environnements

Le traitement complet s'appuie sur une architecture de calcul hybride. Le calcul odométrique lourd (algorithme DLIO) nécessite un environnement Docker isolé pour exécuter les nœuds ROS 2 de manière stable. Les étapes d'alignement manuel interactif et d'extraction topographique s'exécutent directement sur la machine hôte via l'environnement Conda local. Les dépendances Python locales doivent être installées à l'aide du fichier de prérequis.

Commande d'installation locale :
`pip install -r requirements.txt`

## 2. Traitement Odométrique Primaire (DLIO)

Les archives brutes issues de l'expédition matérielle doivent d'abord être converties en trajectoires spatiales et en nuages denses. Cette étape initiale s'effectue dans le conteneur Docker. Il faut exécuter le script d'orchestration `relance_dlio_docker.sh` sur chaque fichier natif capturé sous terre. Les dossiers résultants, identifiables par leur suffixe, possèderont alors les canaux odométriques obligatoires pour la suite des opérations.

## 3. Alignement Séquentiel avec Pegar.py 

(on a appelé Pegar.py...en hommage aux grottes d'El Penon, pour lesquelles ce module a initialement été développé)

Le script Pegar.py a pour fonction de corriger les ruptures de continuité entre plusieurs sessions d'enregistrement consécutives. Il faut placer les archives traitées par DLIO dans le répertoire de travail et exécuter la commande `python3 Pegar.py`. L'application déploiera un serveur web interactif accessible sur le navigateur local au port 5000. 

L'opérateur doit utiliser cette interface WebGL pour ajuster rigoureusement les six degrés de liberté spatiaux, afin de superposer visuellement les fragments de la galerie. L'algorithme se charge ensuite de sauvegarder ces matrices d'orientation dans des fichiers textes de sécurité, puis d'appliquer cette transformation mathématique sur la totalité du format binaire ROS 2. Le script exporte un modèle visuel tridimensionnel global ainsi qu'un dossier centralisé nommé `reseau_optimise_bag`.

## 4. Extraction Topographique avec lidar_to_vtopo.py

L'étape finale exploite le script lidar_to_vtopo.py pour traduire l'archive unifiée en données lisibles par le logiciel de dessin topographique. Il faut lancer la commande `python3 lidar_to_vtopo.py reseau_optimise_bag` dans le terminal de l'environnement local. 

L'algorithme parcourt la nouvelle base de données pour reconstruire l'arbre spatial du nuage de points. Il isole ensuite des coupes perpendiculaires le long de la trajectoire marchée pour calculer mathématiquement l'éloignement des parois (droite, gauche, plafond, sol). Le processus achève la chaîne de traitement en générant un fichier avec l'extension `.tro`, prêt à être injecté et affiché dans l'interface Visual Topo via le serveur graphique virtuel X11.
